# 1002 版本功能文档

## 版本定位

`1002_AI_Sign_Phrase_Assistant` 是独立的手语辅助沟通终端版本。它不再以灯控为核心，而是把 STM32N647 的摄像头、NPU、LCD、USART1 和板载 LED 组织成一个 10 个常用手语短语识别系统。

## 功能链路

```text
camera frame
-> palm detection
-> hand landmarks 21 points
-> sign phrase rule parser
-> candidate phrase + hold progress
-> confirmed phrase
-> LCD display + USART1 JSONL + LED pulse
```

## 新增功能

| 模块 | 功能 |
| --- | --- |
| 手语短语枚举 | `APP_SIGN_G01` 到 `APP_SIGN_G10` |
| 规则识别 | 基于 21 点判断手指伸展、OK 捏合、拇指方向、电话手等 |
| 稳定确认 | 候选手势连续保持 1200ms 后确认 |
| 防刷屏冷却 | 确认后 1500ms 冷却，同一手势长按只输出一次 |
| LCD 状态 | 显示候选 ID、手型、保持百分比、确认结果、冷却状态，使用 ASCII 安全文本 |
| JSONL 事件 | USART1 输出 `sign_confirmed` 事件 |
| 成功反馈 | 板载 LED A 短脉冲提示 |
| 扩展预留 | 文档定义录入模式和用户模板后续路线 |
| 启动诊断 | USART1 输出 `boot` 事件，方便判断应用是否跑起来 |

## USART1 JSONL

串口参数：`115200 8N1`。

输出示例：

```json
{"t":321,"ver":1002,"event":"boot","hold_ms":1200,"cooldown_ms":1500}
{"t":123456,"ver":1002,"event":"sign_confirmed","id":"G10","phrase":"紧急情况，请帮忙","gesture":"Four Fingers","hold_ms":1200,"cooldown_ms":1500}
```

字段说明：

| 字段 | 含义 |
| --- | --- |
| `t` | `HAL_GetTick()` 时间戳 |
| `ver` | 项目版本号 |
| `event` | `boot` 或 `sign_confirmed` |
| `id` | G01 到 G10 |
| `phrase` | 中文输出短语 |
| `gesture` | 当前规则识别到的手型名称 |
| `hold_ms` | 确认保持时间 |
| `cooldown_ms` | 防重复输出冷却时间 |

LCD 端默认字库通常只支持 ASCII，因此屏幕上显示 `G10 EMERGENCY` 这类安全文本；中文短语通过 USART1 的 `phrase` 字段输出。

## 手势定义

| ID | 中文短语 | 规则定义 | 演示建议 |
| --- | --- | --- | --- |
| G01 | 你好 | 拇指、食指、中指、无名指、小指全部伸展 | 五指自然张开 |
| G02 | 谢谢 | 拇指和食指距离很近，中指/无名指/小指伸展 | OK 手势 |
| G03 | 我需要帮助 | 四个长手指收拢 | 握拳，手背朝摄像头更稳 |
| G04 | 我不舒服 | 拇指向下，其余手指收拢 | 拇指竖直向下 |
| G05 | 请稍等 | 只有食指伸展 | 竖起食指 |
| G06 | 我想喝水 | 食指/中指/无名指伸展，小指收拢 | 三指手势 |
| G07 | 请联系家人 | 拇指和小指伸展，食指/中指/无名指收拢 | 电话手势 |
| G08 | 是的 | 拇指向上，其余手指收拢 | 点赞 |
| G09 | 不是 | 食指和中指伸展，无名指/小指收拢 | V 手 |
| G10 | 紧急情况，请帮忙 | 食指/中指/无名指/小指伸展，拇指收起 | 四指张开，区别于 G01 |

G01 和 G10 最容易混淆：G01 要拇指明显张开，G10 要拇指收起。现场演示时可以先展示 LCD 上的 `Cand Gxx` 变化，再保持 1.2 秒触发确认。
